INPUT PERSONAL GENOMES

The input of the PGB is personal genome variants file. The PGB accepts Bgzip/Tabix (1) compressed/indexed VCF (2) and GVF (3) files. User can either provide accessible data ULR link or locally upload data and index files. Using URL link is recommended. Because the volume of files can be locally uploaded is limited up to 20MB due to network condition and server storage. The PGB guaranty input data can only be accessed and view by the owner. The PGB will NOT record any information of user data. The uploaded files will be automatically deleted as soon as the session expired.

Users can also browse built-in genomes for comparison. We currently hold over a thousand individual genomes in the PGB server. Most of them are downloaded from 1000 genome project website (4).

INTERPRETATION OF OUTPUT RESULTS

• Bird's eye view

Bird's eye view is a clock-like personal genome view. Inner circle indicates chromosomes. When a chromosome is selected, the cytobands of selected chromosome are displayed in outer circle. Functional variants distribution is shown by heatmap in which color gradients reflect number of functional significance of the variants in the region. User can scan functional variants in whole genome or selected chromosome/cytoband region. Typically it may take several seconds to scan a cytoband. However, the scanning time varies due to the network condition and length of region.

• Detail view

User can click on displayed objects (variants, genes, ...) to open the detail view of them.
For personal genome variants, the PGB query dbSNP database based on variant position, and merge dbSNP (5) record into the original information of the variants files.
For 'personal genes', the PGB highlights amino acid changes and neighborhood broken TF binding motifs caused by personal genome variants.
Clicking on top-right corner link will redirect to the annotation source.

• Browser view

Personal genome panel displays the input genome variants and their functional consequences. As long as the variant icons are displayed, functional variants are highlighted by purple variant icons. The amino acid changes caused by the personal genome variants are real-timely analyzed and annotated based on gene annotations. Disease and phenotypes that related to personal genome variants or changed genes are displayed below.

PROCESSING METHOD

Basically the PGB map personal genome variants to genes and functional elements, and then map those to diseases, phenotypes and drug interactions, which is the standard approach of variant annotation (6). Various annotations and knowledge bases, such as dbSNP , RefSeq Gene, UCSC knownGene, ensembl Gene, OMIM, GwasCatalog, PharmGKB, etc., are integrated in the PGB for personal genome variant analysis and annotation (5, 7-11).

Particularly, we developed a fast, rule-based approach for annotating transcriptional and translational implications of personal genome variants based on the variant position/type and gene annotations. The PGB also integrates SIFT (12) and PolyPhen2 (13) scores for functional variants evaluation. The scores of personal genome variants are real-timely retrieved from dbNSFP (14) databses by ANNOVAR (15) tool.

SETTINGS

• Tracks in reference genome panel can be managed by checkboxes below the reference genome panel.

• To manage tracks displayed in personal genome panel, click the 'Setting' button on personal genome panel controls.

• User can upload custom tracks to reference genome panel via the 'Upload Custom Track' window. Only URL accessible data can be linked into the PGB.
